# Transparencia de modelo y autoanálisis

Héctor OS muestra proveedor, modelo, nivel y fallback de cada respuesta nueva. Los endpoints `/api/system/model` y `/api/system/report` consultan registros reales de uso y el manifiesto versionado del sistema. La autoevaluación ejecuta la suite existente y produce un prompt corto de mejora basado en pruebas fallidas.
